# PHASE 4：EPUB変換＆ファイル整備

- **原稿ファイル**：`04_本文原稿.md`（※pandocコマンド例では `manuscript.md` として記載）
- **タイトル**：スマホで詐欺に遭わない本 ― シニアが今すぐやるべき7つの設定
- **著者名（ペンネーム）**：村上 健一
- **言語**：日本語（ja）
- **表紙画像**：`cover.jpg`（1600×2560px）

---

## 1. pandocコマンド（メタデータ付き）

### EPUB生成（KDP入稿用・メイン）

```bash
pandoc 04_本文原稿.md \
  -o スマホ詐欺_KDP入稿用.epub \
  --metadata title="スマホで詐欺に遭わない本 ― シニアが今すぐやるべき7つの設定" \
  --metadata author="村上 健一" \
  --metadata lang="ja" \
  --toc --toc-depth=1 \
  --epub-cover-image=cover.jpg
```

- `--toc --toc-depth=1` … `#`（H1）見出しから目次（nav）を自動生成。本書は各章が `#` なので章単位の目次になります。
- `--epub-cover-image=cover.jpg` … 表紙をEPUBに内包します。

### DOCX生成（バックアップ用）

```bash
pandoc 04_本文原稿.md \
  -o スマホ詐欺_KDP入稿用.docx \
  --metadata title="スマホで詐欺に遭わない本 ― シニアが今すぐやるべき7つの設定"
```

### pandocのインストール（未導入の場合）
```bash
# macOS（Homebrew）
brew install pandoc
# Windows（Chocolatey）
choco install pandoc
# Ubuntu/Debian
sudo apt-get install pandoc
```

## 2. でんでんコンバーター使用手順（pandocが使えない場合の代替）

「でんでんコンバーター」は、Markdownをブラウザ上でEPUBに変換できる無料の日本語サービスです。

1. **でんでんマークダウン記法に微調整**
   - 本書の `#`/`##`/`###` 見出しはそのまま使えます。
   - 改ページを入れたい箇所に、独立行で `===`（または各章頭で見出しを使う）。本書は章頭が `#` 見出しなので自動でまとまります。
2. ブラウザで「でんでんコンバーター」を開く
3. `04_本文原稿.md` の中身をテキスト欄に貼り付け、または `.md` をアップロード
4. **タイトル / 著者名**を入力
5. **表紙画像**として `cover.jpg` をアップロード
6. 「目次を自動生成」にチェック →「変換」をクリック
7. 生成された `.epub` をダウンロード

> ヒント：でんでんコンバーターは日本語の縦書き・ルビにも対応。シニア向け実用書は横書きのままでOKです。

## 3. EPUB動作確認手順（Calibreでのプレビュー）

1. 無料ソフト **Calibre** をインストール（calibre-ebook.com）
2. Calibreを起動し、生成した `.epub` を画面にドラッグ＆ドロップで追加
3. 本を選んで「**表示（View）**」をクリック → 内蔵ビューアで開く
4. 次を確認：
   - 最初に**表紙**が表示されるか
   - 左側または目次ボタンで**章ごとの目次（序章〜終章）**が出るか
   - 本文の**日本語が文字化けしていない**か
   - 見出しの大きさ・段落が整っているか
5. （任意）スマホの **Kindleアプリ**に「パーソナル・ドキュメント送信」で送り、実機での文字サイズも確認

## 4. よくあるEPUBエラーと修正方法

| 症状 | 原因 | 修正方法 |
|------|------|----------|
| 日本語が文字化け（□や?） | 文字コードがUTF-8でない | 原稿を**UTF-8**で保存し直す（メモ帳/エディタの保存時に選択） |
| 表紙が表示されない | `cover.jpg` のパス間違い | コマンドと同じフォルダに `cover.jpg` を置く／パスを絶対パスに |
| 目次（nav）が出ない | 見出しレベルの指定ミス | `--toc --toc-depth=1`（または2）を付ける。章頭が `#` か確認 |
| KDPで表紙が小さい/粗いと警告 | 表紙の解像度不足 | 表紙を**1600×2560px以上**で再書き出し |
| 画像がEPUBに入らない | 画像の相対パスが違う | 画像は `images/` に入れ、原稿内のパスを一致させる |
| KDPアップロードで弾かれる | EPUBの構造エラー | Calibreで一度開いて「変換」し直すと整うことが多い |

> KDPは `.epub` と `.docx` のどちらでも入稿可能。**まずEPUBで入稿し、うまくいかなければDOCXを試す**のが確実です。
